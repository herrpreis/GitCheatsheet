Git Cheatsheet

Git Commands

# Command Meaning
1 git init Creates a new Git repository
2 git status inspects the contents of the working directory and staging area
3 git add adds files from the working directory to the staging area
4 git diff shows the difference between the working directory and the staging area
5 git commit permanently stores file changes from the staging area in the repository
6 git log shows a list of all previous commits
7 command Intstruction missing

Git Workflow

# Step How to...
1 Create a branch Get a copy of the code files to work on
2 Add commits Editing, adding, deleting files
3 Pull Requests Request your changes to be added
4 Discuss and review ...your code with teammates
5 Deploy ...from your branch for final testing
6 Merge ...your code into the main branch

GitHub on VC Studio

# Step What to do
1 Do changes Do changes on your file in VS Code, and save them. The update will appear under 'Source Control' (left nav, third from top).
2 Stage Changes In the Source Control tab, the file is listed under 'Changes'. Click the “+” icon to stage the changes. The file will be listed under 'Staged Changes'.
3 Double-check staged changes By clicking the file under 'Staged Changes', you can check if you're happy with the changes.
4 Commit Staged Changes In the message field (about 'Stages Changes'), enter a description for this commit. Then click the checkmark icon above! The new file will be listed at the blue bottom row, next to the arrow up icon (indicating commits that have not yet been pushed).
5 Push your commit Click the '...' in the header, and select Push. Once the commit has been successfully pushed to the server you will notice that up and down arrows have now disappeared from the bottom of VScode
6 Confirm on GitHub If you want, you can check on GitHub wheather your changes appear.
